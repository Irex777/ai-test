🔬 Scoring Methodology

Each app is statically analyzed (regex/parsing of the HTML/CSS/JS) and graded on a scale across 6 dimensions. Scoring is intentionally strict: a typical model lands 40-75/100, not 90+. Speed bonus is computed separately from recorded timings.

🎯 Functionality & Correctness 30 pts

Core mechanics working end-to-end, completeness vs the spec, and edge-case handling.

  • Core mechanics (10): real event handlers wired to working logic, game loops present.
  • Completeness (10): fraction of requested spec features implemented with code.
  • Edge cases (10): try/catch, validation, divide-by-zero guards, empty-state handling.

🏗️ Code Quality & Architecture 20 pts

How well the code is structured, defect-free, and maintainable.

  • Structure (7): function decomposition, no god-functions, meaningful names.
  • Defect avoidance (7): penalize inline onclick=, eval(), innerHTML with input, leaked timers.
  • Maintainability (6): comments, CSS custom properties, DRY, no magic numbers everywhere.

✨ UX & Interaction Polish 20 pts

Does the app feel responsive, give feedback, and respect keyboard users?

  • Micro-interactions (7): CSS transitions, keyframes, transforms, real hover states.
  • Feedback (7): toasts, dialogs, loading states, success/error messaging.
  • Keyboard & input (6): keyboard shortcuts, inline validation, undo, debounce.

🎨 Visual Design 15 pts

Deliberate type system, cohesive color palette, and systematic spacing.

  • Typography (5): font stack, size scale, weight contrast, line-height.
  • Color & cohesion (5): CSS variables, intentional accent colors (not random hex).
  • Layout & spacing (5): grid/flex quality, 8px spacing rhythm, box-shadow depth.

♿ Accessibility 10 pts

The great differentiator — most models score near 0 here.

  • Semantic HTML (4): <header>, <main>, <button> not <div onclick>, heading hierarchy.
  • ARIA & labels (3): aria-label, aria-live, role=, icon-only button titles.
  • Keyboard nav & contrast (3): :focus-visible, tabindex, prefers-reduced-motion.

⚡ Performance & Efficiency 5 pts

File size vs feature set, and runtime discipline.

  • Bundle (3): <15KB efficient; >50KB bloated; large inline base64 = penalty.
  • Runtime (2): requestAnimationFrame over setInterval, debouncing, cleanup.

⏱️ Speed Bonus +5 max

Separate from quality. Per test, fastest of the timed models gets +5, scaled to +0.5 for slowest. Models without recorded timing data get no bonus.
