📖 Methodology

TL;DR

We gave 4 AI models the same 11 prompts, asked each to generate a complete HTML application in a single shot. Then ran 464 automated functional test cases in headless Chromium to score them. No human editing. No iteration.

Fairness note on invocation method: We attempted to run all 4 models through claude -p (Claude Code CLI) for a perfectly fair test. Both GLM and Qwen have Anthropic-compatible API endpoints (/v1/messages). However, claude -p --max-turns 1 returns "Error: Reached max turns (1)" for non-Anthropic models — the CLI is an agent wrapper that expects Claude's tool-use protocol. Sonnet and Opus were called via claude -p; GLM and Qwen via direct API. This is a limitation of the test setup.

Scoring System

Inspired by industry benchmarks:

  • SWE-bench: % of issues resolved via unit test pass rate
  • HumanEval: pass@k — does the code execute correctly?
  • BigCodeBench: Calibrated Pass@1 on executable test cases
  • Aider Polyglot: 225 exercises, test case correctness

Our Method

  1. 464 functional test cases (10-14 per test app) written as JavaScript boolean expressions
  2. Each HTML file loaded in headless Chromium via Playwright
  3. Each test case evaluated in the page context — pass/fail, no subjective scoring
  4. Speed bonus: 0-2 points per test based on generation time ranking (fastest = 2.0, slowest = 0.5)
  5. Total: 138 points maximum (116 functional + 22 speed)

Exact Prompts

📋 Kanban Board

Create a complete, self-contained Todo Kanban board application as a single HTML file (inline CSS + JS, no external dependencies). Requirements: Three columns (To Do, In Progress, Done). Add new tasks with title, description, priority (Low/Medium/High). Drag and drop between columns. Dark theme with CSS variables. LocalStorage persistence. Search/filter tasks. Responsive design. Task count per column. Timestamps. Delete and edit tasks. Animations on drag. Output ONLY raw HTML starting with .

📊 Analytics Dashboard

Create a complete analytics dashboard as a single HTML file (inline CSS + JS, no external dependencies). Requirements: KPI cards with animated counters (Revenue $847K, Users 12.4K, Orders 3.2K, Growth +24%). Revenue chart using SVG/Canvas. Recent activity feed (10 items). Dark theme. Responsive grid layout. Auto-refresh timer (simulated data updates). Date range filter. Export button. Hover effects on cards. Output ONLY raw HTML starting with .

♟ Chess Game

Create a chess game as a single HTML file with inline CSS+JS, no external dependencies. Requirements: 8x8 board with Unicode pieces. Click piece to select, click square to move. Turn indicator (white/black). Dark theme. New Game button. Captured pieces display. Move history. Highlight selected piece and valid moves. Check detection with visual indicator. Output ONLY raw HTML starting with .

Test Runner

All test cases are defined in test_cases.json and executed via:

from playwright.sync_api import sync_playwright
with sync_playwright() as p:
    browser = p.chromium.launch(headless=True)
    page = browser.new_page()
    page.goto("file://test_output.html")
    result = page.evaluate(test_case_js)  # returns True/False

No human edited any output. All 44 files are single-shot generations.

Test suite: 464 test cases across 11 applications, run via Playwright in headless Chromium.
